# 研行记 · 模型推荐与配置

后端通过环境变量 `OPENAI_API_KEY`、`OPENAI_BASE_URL`、`OPENAI_MODEL` 调用 **OpenAI 兼容 API**，可对接多种多模态/长上下文模型。

---

## 一、单模型推荐（按场景）

| 模型 | 读图/读表/多模态 | 长上下文 | 速度 | 质量 | 适用场景 | 配置示例 |
|------|------------------|----------|------|------|----------|----------|
| **GPT-4o** | ⭐⭐⭐⭐⭐ | 128K | 快 | 高 | 综合最强，多模态、表格、长文档 | `OPENAI_MODEL=gpt-4o`，Base URL 用 OpenAI 或 OneAPI |
| **GPT-4o-mini** | ⭐⭐⭐⭐ | 128K | 很快 | 中高 | 性价比高，当前默认 | `OPENAI_MODEL=gpt-4o-mini` |
| **Claude 3.5 Sonnet / 3.7** | ⭐⭐⭐⭐⭐ | 128K | 快 | 高 | 文档/表格/结构化输出强 | 需支持 Claude 的 Base URL + 对应 model 名 |
| **DeepSeek R1 / V3** | ⭐⭐⭐⭐ | 64K | 很快 | 高 | 中文+推理好、成本低 | `OPENAI_MODEL=deepseek-chat` 等，Base 用 DeepSeek 或 OneAPI |
| **Gemini 1.5 Pro** | ⭐⭐⭐⭐⭐ | 100 万 token | 中 | 高 | 超长文档、多模态 | 需 Google AI 或提供 Gemini 的兼容 API |
| **Qwen2.5-Max** | ⭐⭐⭐⭐ | 128K | 中 | 高 | 中文场景优化 | 需通义千问或兼容 API |

- **读图读表科研能力更强**：优先 **GPT-4o**、**Claude 3.5/3.7**、**Gemini 1.5 Pro**（若后续接入图片输入，这些多模态优势更大）。
- **生成更快、质量高、成本可控**：**DeepSeek R1/V3**、**GPT-4o-mini**。
- **上下文要强**：**Gemini 1.5 Pro**（百万级）、**GPT-4o / Claude**（128K）。

## 二、.env 配置示例

在项目根目录的 `.env` 中设置（保持与现有 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 一致，只改模型名即可）：

```bash
# 使用 GPT-4o（OpenAI 官方）
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o

# 使用 DeepSeek（需有 DeepSeek 或 OneAPI 的 base_url 和 key）
# OPENAI_BASE_URL=https://api.deepseek.com/v1
# OPENAI_MODEL=deepseek-chat

# 使用国内 OneAPI/统一转发（替换为你的 base_url 和 key）
# OPENAI_BASE_URL=https://your-oneapi-host/v1
# OPENAI_MODEL=gpt-4o
# 或 OPENAI_MODEL=deepseek-chat 等，依 OneAPI 支持的模型名填写
```

修改后**重启后端**（`uvicorn main:app --host 127.0.0.1 --port 8002`）生效。

## 三、混搭使用思路（可选）

若希望「质量更高」或「成本更优」，可考虑两阶段（需改代码）：

1. **阶段一（理解 + 抽取）**：用**长上下文/多模态强的模型**（如 GPT-4o、Claude、Gemini）处理转写+PPT+文献全文，输出结构化草稿或证据列表。
2. **阶段二（润色/精简）**：用**更快、更便宜的模型**（如 DeepSeek、GPT-4o-mini）在草稿基础上做润色、压缩或二次整理。

当前研行记为**单模型单次调用**；若你后续要加「混搭」，可在 `main.py` 的 `process_meeting` 里先调用模型 A 得到中间结果，再调用模型 B 生成最终 JSON。

## 四、说明

- 当前请求只传**文本**（转写、PPT 文本、文献文本），未传 PDF/PPT 的**图片**。若未来改为同时传图，**GPT-4o、Claude、Gemini** 等多模态模型在「读图读表」上会更有优势。
- 表中「读图/读表」在仅文本输入时，主要体现在模型对长文本、表格描述、结构化输出的能力；接入图片后，多模态能力会真正发挥出来。
